📬 cloud-mail-worker

Cloudflare Email → 飞书群机器人 · 部署指南

🏗️ 架构概览

发件人
  └─✉️ Email 发送到 your@domain.com
       └─☁️ Cloudflare Email Routing
            └─⚙️ Cloudflare Worker (cloud-mail-worker)
                 ├─ 1. 发件人过滤（白/黑名单）
                 ├─ 2. 解析邮件（发件人/主题/正文）
                 └─ 3. POST 到飞书 Webhook
                      └─💬 飞书群机器人消息

📋 前置条件

  • ☁️ Cloudflare 账号 — 在 Cloudflare 管理你的域名
  • 🌐 域名 — DNS 由 Cloudflare 托管
  • 💬 飞书群 — 有权限添加群机器人
  • 🖥️ Node.js — 安装 node -v ≥ 18

🚀 部署步骤

1

获取飞书群机器人 Webhook

  1. 打开目标飞书群 → 点击右上角 ··· → 群设置
  2. 进入 群机器人 → 点击 添加机器人
  3. 选择 自定义机器人（通过 Webhook 发送消息）
  4. 设置机器人名称（如「邮件通知」），点击「完成」
  5. 复制 Webhook 地址，格式如下：
https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
💡 安全建议： 在飞书机器人设置中开启 IP 白名单， 将 Cloudflare 出口 IP 段 加入白名单。
2

克隆项目 & 安装依赖

# 进入项目目录
$ cd cloud-mail-worker

# 安装依赖（wrangler + @cloudflare/email）
$ npm install
3

配置飞书 Webhook（加密存储）

$ npx wrangler secret put FEISHU_WEBHOOK_URL
# 粘贴你的飞书机器人 Webhook URL，按 Enter
# 输出: ✨  Success! Uploaded secret FEISHU_WEBHOOK_URL

# （可选）转发到第二个群
$ npx wrangler secret put FEISHU_WEBHOOK_URL_2
⚠️ 重要： 使用 wrangler secret 而不是写在 wrangler.toml 中， Webhook URL 会被加密存储，不会泄露。
4

修改域名配置

编辑 wrangler.toml，将域名改为你自己的：

# wrangler.toml 第 24 行
[[email.routing]]
  pattern = "*@你的域名.com"
  action = "worker"

如果你的 Worker 名称不叫 cloud-mail-worker，也一并修改 name 字段。

5

部署 Worker

$ npx wrangler deploy

# 输出类似：
Total Upload: xx KB
Uploaded cloud-mail-worker (xx sec)
Published cloud-mail-worker (xx sec)
  https://cloud-mail-worker.xxxx.workers.dev
  Email: cloud-mail-worker
6

配置 Email Routing 路由规则

  1. 登录 Cloudflare Dashboard
  2. 选择你的域名 → 左侧 Email → Email Routing
  3. 如果首次使用，按引导完成域名验证（添加 MX 记录等）
  4. 进入 Routing rules 标签页，点击 Create rule
  5. 填写：
    • Action: Send to Worker
    • Worker: cloud-mail-worker
    • Catch-all: 勾选（或填写具体地址如 contact@yourdomain.com）
  6. 点击 Save
💡 提示： 你也可以在 Catch-all address 中设置， 将所有发往该域名的邮件都转发到 Worker。

向你的域名邮箱发送一封邮件，然后查看飞书群是否有通知。

方法三：手动触发测试消息

先设置 AUTH_TOKEN：

$ npx wrangler secret put AUTH_TOKEN
# 输入一个随机 token，如 my-test-token-123

然后发送测试请求：

$ curl -X POST https://cloud-mail-worker.xxxx.workers.dev/test \
  -H "X-Auth-Token: my-test-token-123" \
  -H "Content-Type: application/json"

$ npx wrangler tail
# 实时查看邮件处理日志

⚙️ 定制化配置

所有配置通过环境变量控制，无需修改代码：

变量名 必填 说明 示例
FEISHU_WEBHOOK_URL 是 飞书群机器人 Webhook URL https://open.feishu.cn/...
FEISHU_WEBHOOK_URL_2 否 第二个飞书群 https://open.feishu.cn/...
ALLOWED_SENDERS 否 白名单，逗号分隔，支持 *@domain boss@corp.com,*@team.com
BLOCKED_SENDERS 否 黑名单，逗号分隔 spam@bad.com
FEISHU_MSG_TITLE 否 飞书消息标题 📬 新邮件提醒
FEISHU_MSG_TAG 否 消息前缀标签 [紧急]
FEISHU_USE_CARD 否 设为 true 使用消息卡片格式 true
FEISHU_MAX_BODY_PREVIEW 否 正文预览最大字符数 500
LOG_LEVEL 否 日志级别 debug
AUTH_TOKEN 否 测试接口鉴权 my-token

非敏感变量（白名单、标题等）也可以写在 wrangler.toml 的 [vars] 中：

[vars]
ALLOWED_SENDERS = "friend@example.com"
FEISHU_MSG_TITLE = "📬 新邮件通知"
LOG_LEVEL = "info"

🔧 常见问题

❓ 收不到飞书通知？
  1. 运行 npx wrangler tail 看日志是否有报错
  2. 确认 FEISHU_WEBHOOK_URL 已正确设置
  3. 确认域名已完成 Email Routing 配置（MX 记录必须生效）
  4. 检查发件人是否被过滤规则拦截
  5. 在飞书群中 @机器人 确认它是否在线
❓ 飞书返回 code: 10003？

Webhook URL 不正确或已失效。在飞书群设置中重新生成 Webhook 地址。

❓ 邮件正文乱码？

Worker 默认支持 UTF-8、Base64、Quoted-Printable 解码。如果仍乱码，可能是非 UTF-8 编码邮件（如 GB2312），需要修改 src/email-parser.js 中的解码逻辑。

❓ 部署时权限错误？

确保已用 npx wrangler login 登录 Cloudflare 账号。

📂 项目文件结构

cloud-mail-worker/ ├─ src/index.js — 主入口（邮件处理 + HTTP 接口） ├─ src/email-parser.js — 邮件解析（RFC 2822 / MIME） ├─ src/feishu.js — 飞书机器人 API 封装 ├─ wrangler.toml — Cloudflare Worker 配置 ├─ package.json — 依赖和脚本 ├─ .env.example — 环境变量示例 ├─ .gitignore └─ README.md — 完整文档

🧠 技术要点

邮件如何处理？

  1. Cloudflare Email Routing 收到发给你的域名的邮件
  2. 根据路由规则，邮件转发给 cloud-mail-worker
  3. Worker 的 email() 事件被触发，收到 EmailMessage 对象
  4. 从 message.raw（ReadableStream）读取原始 RFC 2822 邮件内容
  5. email-parser.js 解析出发件人、收件人、主题、正文（支持 multipart）
  6. feishu.js 将解析结果格式化为富文本消息
  7. POST 到飞书 Webhook 端点，机器人将消息发到群中

飞书消息格式

默认使用 富文本（post） 格式，显示清晰、兼容性好。

设置 FEISHU_USE_CARD=true 可切换为 消息卡片（interactive） 格式，视觉效果更佳（带颜色 header、分割线等）。
